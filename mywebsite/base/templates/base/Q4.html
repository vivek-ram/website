TDD implementation

TDD means “Test Driven Development”. The primary goal of TDD is to make the code clearer, simple, and bug-free. Writing tests after writing the code is hard. As the code can be written without testability in mind, it might not be testable. Hence, we would need to change our code again to make it testable and I started with some tutorial about TTD in Django. they are two types of tests one is a functional test and another is a unit test. So the functional test is all about the deal with the way a user will interact with your project also known as functionality on other hand unit test is all about Ensure that small pieces of your project are working as they should. And writing the tests in selenium is a really easy one it allows us to test it on a specific browser.
